Total siempre visible

El total de la orden permanece fijo en el header durante toda la pantalla S-07 — el usuario nunca pierde de vista cuánto debe cubrir sin importar cuánto haga scroll.

El nombre del cliente también está en el header para confirmar el contexto sin ambigüedad.

Modo Parcialidades

Pantalla 1 3 parcialidades mensuales. Roberto ajustó la fecha de la 1.ª manualmente.

El sistema calcula montos equitativos (total / n). Si el usuario edita un monto, el sistema redistribuye el delta en las parcialidades restantes.

La suma de parcialidades debe igualar el total para habilitar "Confirmar Pago". El check verde confirma que cuadra.

Si no cuadra: indicador rojo con el delta faltante y botón deshabilitado.

Constructor de Métodos de Pago

Layout por fila: dropdown [minWidth=120dp] · campo monto [weight=1f] · botón × [wrapContent]. Esta distribución soporta fontScale elevado sin colapsar los controles.

El botón × lleva contentDescription: "Eliminar método de pago [tipo]" — p.ej. "Eliminar método de pago Efectivo".

"Efectivo" no puede repetirse. Si ya existe una fila de Efectivo, el dropdown lo oculta de las opciones de filas adicionales.

Restante por asignar (LiveRegion)

El contador usa LiveRegionMode.Polite: TalkBack lo anuncia automáticamente cuando cambia, sin que el usuario mueva el foco.

Ámbar Restante > $0.00 — botón "Confirmar" deshabilitado.

Verde Restante = $0.00 — botón "Confirmar" habilitado.

Crédito a Favor

Si el cliente tiene Crédito a Favor, aparece el chip informativo (color cian tonal) con el monto y el botón "Aplicar". Al aplicarlo, se agrega como método de pago automáticamente y el Restante se actualiza.

El cian está en el chip solo como fondo tonal — no como texto ni ícono de sync, respetando la reserva exclusiva del token sync-ok.
