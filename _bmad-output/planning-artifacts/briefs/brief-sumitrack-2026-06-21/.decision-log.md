# Decision Log — Sumitrack Product Brief

## Sesión: 2026-06-21

### Decisiones registradas

| # | Decisión | Razonamiento | Alternativas rechazadas |
|---|----------|--------------|------------------------|
| 1 | MVP = Android + API + DB en la nube (sin web portal) | El proveedor opera en campo; la web es un deseable a futuro | Web portal en MVP — descartado por prioridad |
| 2 | Arquitectura multi-tenant desde el inicio | Josemtz tiene experiencia con sistemas de 30k+ negocios; visión SaaS futura justifica el costo inicial | Single-tenant primero — descartado para evitar refactors costosos |
| 3 | Offline-first con SQLite + resync | El proveedor opera en campo con conectividad variable en Yucatán | Asumir conectividad — descartado por riesgo operativo |
| 4 | Notificaciones vía push (v1) | Suficiente para el MVP; integración con Calendar es deseable pero fuera de spec | SMS/WhatsApp/email — fuera de MVP |
| 5 | Nota de venta = ticket impreso (Bluetooth) + opción email | No se requiere CFDI para MVP; ticket contiene información fiscal del proveedor como dato de control | CFDI desde día uno — descartado para MVP, considerado en visión |
| 6 | Pago en exhibición o parcialidades | Es el modelo de negocio central del proveedor (venta a crédito) | Solo pago en una sola exhibición — no cubre el caso de uso real |
| 7 | Stack: Android (Kotlin) + .NET API + Cloud DB | Alineado con experiencia de 12+ años de Josemtz; stack conocido reduce riesgo | Stack alternativo — no discutido; se asume por background |

### Preguntas abiertas / diferidas

- Stack cloud específico (Azure, AWS, GCP) — no definido en brief, a decidir en arquitectura
- Modelo de monetización SaaS (freemium, por tenant, por usuario) — diferido a fase de crecimiento
- CFDI/SAT — confirmado fuera del MVP, a diseñar en V2
- Integración con calendario de recordatorios — fuera de spec, deseable futuro
- Migración de datos del sistema en papel — no discutida; probablemente no aplica dado volumen
